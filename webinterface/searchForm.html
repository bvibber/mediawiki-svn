Test web interface for lucene-search 2.0

Highlights:

  • Prefixed searches. If you want to search only some namespace, prefix the search string with it. E.g. help:editing will search only help namespace. You can combine namespaces, e.g. help,help_talk:editing. You can also use aliases (these will be customizable in MW). I've predefined some aliases (the list is below), so the last query can be written: h,ht:editing. You can also search everything by prefixing it with all. E.g. all:editing
  • Category searches. You can limit search by categories (note: this won't work recursivelly for subcategories, you need to include them!). E.g. white cheese incategory:cheeses. To avoid ambiguity, category names need to be given exactly. E.g. monument incategory:"World Heritage Sites in Serbia". By default category intersection is searched, but any combination can be searched using AND/OR and +/- operators.
  • Improved scoring. Short articles are not strongly favored any more, and stemmed words are penalized. Number are included in the index.
  • Accentless search. Accents are automatically removed in both indexed and searched terms.
